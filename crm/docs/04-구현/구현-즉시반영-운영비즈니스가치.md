# 구현 문서: 즉시 반영 + 운영 비즈니스 가치

## 1. 구현 범위
- 즉시 반영: 권한/세션 최신화, 관리자 권한 관리, Soft Delete 복구
- 운영 기능: 운영 헬스 엔드포인트 확대

## 2. 변경 항목

### 2.1 공통
- `src/lib/middleware/auth.ts`
  - Access Token 검증 후 `userId` 기준으로 DB 조회
  - 토큰 payload 역할과 무관하게 현재 DB 역할로 `c.set('user')`
  - 즉시 역할 변경 반영 기반 마련

### 2.2 관리자 권한 관리
- `src/modules/auth/index.ts`
  - `GET /admin/users` 추가
  - `PATCH /admin/users/:id/role` 추가
  - `adminOnly` 적용
  - 관리자만 사용자 목록 조회/역할 변경

### 2.3 Soft Delete 복구
- `src/modules/contacts/index.ts`
  - `POST /:id/restore`
- `src/modules/deals/index.ts`
  - `POST /:id/restore`
- `src/modules/tasks/index.ts`
  - `POST /:id/restore`
- 삭제 데이터만 복구 대상, 소유자·Admin 제어 준수

### 2.4 운영 헬스 확장
- `src/modules/health/index.ts`
  - `GET /ready` 추가: DB 준비 상태 점검 + 장애 시 `503`
  - `GET /live` 추가
  - `GET /metrics` 추가: uptime, memory, db 상태
  - 루트 `/api/v1/health`는 응답 형식 일관성 유지 위해 success wrapper 적용

## 3. 영향도
- 스키마 변경 없음
- OpenAPI 문서는 일부 누락 가능(요청 기준 경로 보완 필요)
- 기존 기능 동작 호환성: 라우트 경로 충돌 없음

## 4. 배포/운영
- 코드 레벨 배포 후 `/api/v1/health/ready`, `/api/v1/health/live`, `/api/v1/health/metrics` 동시 검증
- 관리자 변경 후 즉시 영향 확인:
  1) `/api/v1/auth/admin/users/:id/role` 호출
  2) 동일 Access Token으로 권한 체크 API 재시도
