## 1. 디자인 시스템 개요

### 1.1 디자인 철학

VIVE CRM은 **"바쁜 영업 담당자를 위한 최소 클릭 CRM"**을 목표로 한다. 불필요한 장식을 최소화하고, 핵심 기능에 빠르게 접근할 수 있는 직관적인 인터페이스를 제공한다.

| 원칙 | 설명 | 적용 |
|------|------|------|
| **단순성 (Simplicity)** | 화면당 하나의 주요 목표 | 대시보드, 고객 목록 등 화면 단위로 명확한 목표 부여 |
| **일관성 (Consistency)** | 반복되는 패턴 유지 | 같은 기능은 같은 위치, 같은 색상, 같은 액션 |
| **피드백 (Feedback)** | 모든 사용자 행동에 응답 | 로딩 상태, 성공/실패 알림, 즉각적인 상태 변경 |
| **접근성 (Accessibility)** | WCAG 2.1 AA 준수 | 색상 대비 4.5:1, 키보드 네비게이션, 스크린 리더 대응 |

### 1.2 디자인 시스템 기반

- **스타일링**: Tailwind CSS v3.x
- **컴포넌트 라이브러리**: Radix UI (Headless) + Tailwind로 커스텀 스타일링
- **아이콘**: Lucide React
- **타이포그래피**: Inter (Latin), Noto Sans KR (한글)

### 1.3 컬러 시스템

```css
/* Primary Colors - 브랜드 컬러 */
```

### 1.4 타이포그래피

| 스타일 | 크기 | 굵기 | 용도 |
|--------|------|------|------|
| Page Title | 24px | 600 | 페이지 헤더 |
| Section Title | 20px | 600 | 섹션 제목 |
| Card Title | 18px | 600 | 카드 제목 |
| Body Large | 16px | 400 | 본문 |
| Body | 14px | 400 | 기본 텍스트 |
| Small | 12px | 400 | 보조 텍스트, 라벨 |

### 1.5 스페이싱 시스템

Tailwind 기본 스케일을 사용하되, 4px 베이스 그리드 시스템:

```css
/* spacing */
4px   (1)   /* xs */
8px   (2)   /* sm */
12px  (3)
16px  (4)   /* base */
20px  (5)
24px  (6)   /* lg */
32px  (8)   /* xl */
48px  (12)
64px  (16)  /* 2xl */
```

### 1.6 반응형 브레이크포인트

| 이름 | 너비 | 용도 |
|------|------|------|
| `sm` | 640px | 모바일 대응 |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 데스크톱 |
| `xl` | 1280px | 큰 화면 |

## 4. 공통 컴포넌트 라이브러리

### 4.1 버튼 (Button)

| 변형 | 용도 | 클래스 |
|------|------|--------|
| Primary | 주요 액션 | `bg-primary-600 hover:bg-primary-700 text-white` |
| Secondary | 보조 액션 | `bg-gray-100 hover:bg-gray-200 text-gray-900` |
| Danger | 위험 액션 | `bg-error-500 hover:bg-error-600 text-white` |
| Ghost | 아이콘, 링크 | `hover:bg-gray-100 text-gray-600` |

### 4.2 입력 폼 (Form)

## 6. 반응형 대응

### 6.1 모바일 (sm)

- 사이드바는 햄버거 메뉴로 숨김
- 테이블은 카드 리스트로 전환
- 칸반 보드는 단일 컬럼 스크롤
- 하단 고정 FAB (빠른 등록)

### 6.2 태블릿 (md)

- 사이드바는 접힘/펼침 가능
- 테이블은 가로 스크롤 가능
- 칸반 보드는 2열 배치

### 6.3 데스크톱 (lg+)

- 사이드바 고정 표시
- 전체 화면 활용
- 칸반 보드는 4~5열 배치
