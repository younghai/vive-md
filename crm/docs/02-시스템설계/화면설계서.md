## 1. 디자인 시스템

### 1.1 디자인 철학

- **단순성**: 불필요한 요소 제거, 핵심 기능에 집중
- **일관성**: 동일한 패턴과 컴포넌트 재사용
- **피드백**: 사용자 행동에 대한 즉각적인 시각적 응답
- **접근성**: WCAG 2.1 AA 준수

## 3. 화면 상세 설계

### 3.1 로그인/회원가입 화면

**경로**: `/login`, `/register`

| 요소 | 타입 | 설명 |
|------|------|------|
| 이메일 | Input | email 타입 |
| 비밀번호 | Input | password 타입, 8자 이상 |
| 이름 | Input | 회원가입 시만 |
| 회사 | Input | 선택사항 |
| 제출 버튼 | Button | Primary, 전체 폭 |

**유효성 검증**:
- 이메일: 이메일 형식
- 비밀번호: 8자 이상, 영문+숫자+특수문자

---

### 3.2 대시보드 화면 (/)

**레이아웃**: 그리드 시스템

**구성 요소**:

| 섹션 | 내용 |
|------|------|
| KPI 카드 | 총 고객, 진행중 딜, 미완료 작업, A등급 리드 |
| AI 추천 | 오늘의 추천 행동 상위 5개 |
| 파이프라인 요약 | 단계별 딜 수, 금액 |
| 최근 활동 | 최근 10개 활동 타임라인 |

---

### 3.3 고객 목록 화면

**경로**: `/contacts`

**구성 요소**:

#### 헤더
- 제목 + 고객 수
- 검색 (이름, 이메일, 전화)
- 필터 (등급, 태그)
- 정렬 (최신순, 이름순, 스코어순)
- 추가 버튼

#### 데이터 테이블

| 컬럼 | 설명 |
|------|------|
| 이름 | 고객명 + 회사 |
| 연락처 | 이메일, 전화번호 |
| 등급 | A/B/C/D 배지 + 스코어 |
| 태그 | 태그 리스트 |
| 최근 연락 | 날짜 |
| 작업 | 수정, 삭제 |

#### 상세 패널 (우측 슬라이드)
- 프로필 정보
- 리드 스코어 (점수, 등급, AI 요약)
- 태그 편집
- 메모
- 활동 이력 (타임라인)
- 딜 목록
- 작업 목록

### 3.4 딜 파이프라인 화면

**경로**: `/deals`

**레이아웃**: 칸반 보드 (가로 스크롤)

**단계별 컬럼**:
1. LEAD (Gray)
2. QUALIFIED (Blue)
3. PROPOSAL (Violet)
4. NEGOTIATION (Amber)
5. WON (Green)
6. LOST (Red)

**딜 카드**:
- 제목
- 고객명
- 금액
- 성공 확률
- 마감일
- 우선순위 표시

**인터랙션**:
- 드래그 앤 드롭으로 단계 이동
- 이동 시 이력 자동 기록

---

### 3.5 작업 목록 화면

**경로**: `/tasks`

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 필터 | 전체/진행중/완료/마감임박 |
| 정렬 | 마감일순, 우선순위순 |

**작업 아이템**:
- 체크박스 (완료 토글)
- 제목
- 우선순위 (높음/중간/낮음)
- 마감일 (지난 경우 강조)
- 연결 고객/딜
- 수정/삭제 버튼

---

### 3.6 활동 로그 화면

**경로**: `/activities`

**레이아웃**: 타임라인

**구성 요소**:
- 날짜 구분
- 활동 아이템 (시간, 유형 아이콘, 제목, 고객명)
- 유형: NOTE, CALL, EMAIL, MEETING

**필터**:
- 유형 필터
- 기간 필터
- 고객 필터

---

### 3.7 리포트 화면

**경로**: `/reports`

**탭**:
- 주간 리포트
- 월간 리포트
- 활동 통계

**위젯**:
- 성공률 (계약 체결율)
- 파이프라인 변화
- 리드 스코어 분포 (차트)
- 활동 요약

## 5. 인터랙션 패턴

### 5.1 CRUD 패턴

| 작업 | 패턴 |
|------|------|
| 생성 | 모달/페이지 -> 폼 -> 저장 -> 토스트 |
| 조회 | 클릭 -> 상세 패널/페이지 |
| 수정 | 편집 버튼 -> 인라인/모달 -> 저장 |
| 삭제 | 버튼 -> 확인 -> 삭제 -> 토스트 |

### 5.2 AI 기능

- 리드 스코어: 고객 상세에서 자동 표시, 재계산 버튼
- 추천: 대시보드에서 확인, 클릭 시 해당 고객으로 이동

## 6. 반응형 대응

### 모바일 (< 640px)
- 사이드바: 햄버거 메뉴
- 테이블: 카드 리스트
- 파이프라인: 단일 컬럼

### 태블릿 (768px)
- 사이드바: 아이콘만 표시
- 테이블: 주요 컬럼만

### 데스크톱 (1024px+)
- 사이드바: 전체 메뉴
- 상세 패널: 우측 슬라이드
