### 4.3 데이터 롤백 방법

| 시나리오 | 롤백 방법 | 데이터 영향 | 비고 |
|----------|----------|-------------|------|
| 스키마 변경만 있는 경우 | Prisma 롤백 마이그레이션 실행 | 신규 데이터 보존 | |
| 데이터 변환이 있는 경우 | Supabase Point-in-Time 복원 | 배포 후 입력 데이터 유실 가능 | 유실 데이터 복구 절차 필요 |
| 전체 복원이 필요한 경우 | Supabase 전체 DB 백업 복원 | 배포 시점 이후 전체 데이터 유실 | 최후 수단 |

**데이터 복구 절차** (배포 후 입력 데이터 보존 필요 시):

1. 롤백 전 현재 DB 스냅샷 생성 (Supabase Backup)
2. 이전 버전으로 DB 롤백 (Prisma migrate rollback)
3. 스냅샷에서 배포 후 신규 데이터 추출
4. 추출한 데이터를 롤백된 DB에 적용
5. 데이터 정합성 검증
