# 요구사항 분석: 즉시 반영 + 운영 비즈니스 가치 기능

- `docs/05-테스트/테스트케이스.md`

  - P2 제안: `/health/ready`, `/health/live`, `/metrics`

## 2. 신규 요구사항

### R1. 즉시 반영 기능
1. 권한 변경 즉시 반영
- 기대값: 관리자 권한 변경 후 사용자 토큰 재발급 없이 API 접근 권한이 즉시 변경되어야 함.
- 근거: 권한 정책 변경 테스트(`TC-FN-PERM-006`) 및 비즈니스 정책.

2. 관리자 권한 관리 API
- 관리자만 대상 계정 목록 조회 및 권한 변경 가능.
- 변경 결과는 현재 토큰 기반 권한(요청 당시의 DB 역할값)으로 즉시 적용되어야 함.

3. Soft Delete 데이터 복구
- `contacts`, `deals`, `tasks`는 삭제 플래그만 반영.
- 삭제 대상은 기본 목록에서 제외되고, `/{id}/restore`로 복구 가능.

### R2. 운영 비즈니스 가치 기능
1. 운영 헬스 체인 확장
- `/api/v1/health/ready`: DB 준비 상태 포함.
- `/api/v1/health/live`: 프로세스 생존성.
- `/api/v1/health/metrics`: 가동 지표(메모리/가동시간/DB 상태).

2. 운영 감사/추적 강화(1차)
- 인증/인가 검증은 매 요청 시 DB 최신 역할로 판단해 정책 위반 시 즉시 차단.

## 3. 비기능 요구
- 기존 라우트와 호환되는 스키마 유지.
- 인증 미들웨어는 `/api/v1/auth/{register,login,refresh}`와 헬스 경로는 공개 유지.
- 복구 API는 객체 소유권/관리자 권한을 동일하게 강제.

## 4. 수용 기준
- 관리자 권한 변경 후 동일 Access Token 재사용 시 권한이 즉시 반영되어야 함.
- 삭제 데이터 복구 API 호출 시 상태가 `deletedAt!=null`에서 정상 복귀해야 함.
- `/health/ready`에서 DB 장애 시 503을 반환해야 함.
