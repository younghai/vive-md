# 용어 규칙 (Terminology Convention)

> **이 문서는 본 프로젝트의 모든 문서에서 반드시 준수해야 하는 용어 표기 규칙이다.**
> 문서 작성·수정 시 아래 규칙을 엄격히 따를며, 새로운 용어가 등장할 경우 이 문서에 먼저 등록한 후 사용한다.

|------|------|
| **프로젝트명** | VIVE CRM |
| **문서 버전** | v1.0 |
| **작성일** | 2026-02-24 |
| **최종 수정일** | 2026-02-24 |

## 1. 표기 원칙

| # | 원칙 | 올바른 예 | 잘못된 예 |
|---|------|----------|----------|
| 1 | **영문 약어**는 원문 그대로 표기한다. 한글 음차/번역하지 않는다 | `CRM`, `AI`, `API` | ~~씨알엠~~, ~~에이아이~~ |
| 2 | **첫 등장 시** 약어의 풀네임을 괄호로 병기한다. 이후에는 약어만 사용한다 | "AI(Artificial Intelligence) 기반..." → 이후 "AI" | 매번 풀네임 반복 |
| 3 | **한글 용어**가 확정된 경우 한글을 주 표기로 하되, 첫 등장 시 영문을 괄호로 병기한다 | "리드(Lead)" | ~~리드~~, ~~Lead~~ 혼용 |
| 4 | **동일 개념에 복수 표현 금지.** 하나의 개념에는 하나의 용어만 사용한다 | "고객"으로 통일 | ~~클리언트~~, ~~커스토머~~ 혼용 |
| 5 | **서비스명은 정확한 형태만 사용한다.** 줄임말이나 변형 금지 | VIVE CRM | ~~Vive~~, ~~VIVECRM~~ |
| 6 | **복합어 띄어쓰기**는 용어 사전의 "표기" 열을 따른다 | "리드 스코어" | ~~리드스코어~~ |
| 7 | **우선순위 표기**는 P0/P1/P2만 사용한다 | P0, P1, P2 | ~~높음/중간/낮음~~, ~~Must/Should/Nice~~ |

## 2. 용어 사전

### 2.1 서비스 고유 용어

| ID | 용어 (표기) | 영문 풀네임 | 정의 | 비고 |
|----|------------|------------|------|------|
| T-001 | VIVE CRM | - | 본 프로젝트의 서비스명. AI 기반 고객 관리 플랫폼 | **정식 명칭. 줄임말 금지** |
| T-002 | 리드 | Lead | 잠재 고객. 영업 기회가 있는 개인 또는 기업 | 첫 등장 시 "리드(Lead)" 병기 |
| T-003 | 딜 | Deal | 영업 기회. 특정 고객과의 계약 체결을 목표로 하는 영업 활동 단위 | |
| T-004 | 파이프라인 | Pipeline | 영업 단계(리드 → 기회 → 제안 → 협상 → 계약)의 시각적 흐름 | |
| T-005 | 스테이지 | Stage | 파이프라인의 각 단계 | "단계"로도 사용 가능 |
| T-006 | 리드 스코어 | Lead Score | 고객의 구매 가능성을 수치화한 점수 (0~100) | |
| T-007 | 터치포인트 | Touchpoint | 고객과의 모든 접촉 지점 (이메일, 전화, 미팅 등) | |
| T-008 | 팔로업 | Follow-up | 고객과의 후속 조치 | |
| T-009 | 다음 행동 | Next Action | AI가 추천하는 다음 영업 활동 | |
| T-010 | 활동 | Activity | 영업 활동 (이메일 발송, 전화, 미팅, 메모 등) | |

### 2.4 우선순위 표기

| 표기 | 의미 | 사용 맥락 |
|------|------|----------|
| P0 | Must Have | MVP에 반드시 포함. 이것 없이는 서비스가 성립하지 않음 |
| P1 | Should Have | MVP에 포함 권장. 핵심 경험을 위해 필요하지만 최소 기능은 아님 |
| P2 | Nice to Have | MVP에서 시간이 허용되면 포함. 없어도 핵심 가치 검증 가능 |

## 3. 용어 추가/변경 절차

1. 새로운 용어가 필요한 경우 → **이 문서의 해당 카테고리에 ID 부여 후 등록**
2. 표기 원칙(1절) 적용하여 표기 형태 확정
3. **프로젝트 내 모든 문서에 일관 반영** (기존 문서에 다른 표현이 있으면 수정)
4. 용어 삭제/변경 시 → 변경 이력에 기록 후 전체 문서 일괄 반영
