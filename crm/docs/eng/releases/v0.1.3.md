# Release Notes: v0.1.3

Date: 2026-02-26

## Summary

This release implements P2 features including dashboard enhancements, the notification system, and business rule enforcement.

## Highlights

### Dashboard Enhancements

- **New metrics:** `newContactsThisWeek`, `todayRecommendationsCount` added to the summary
- **`GET /dashboard/activities`:** Recent activities endpoint with a `limit` parameter

### Reports

- **`GET /reports/activities`:** Activity statistics by type
- **Weekly/monthly reports:** Added `successRate`, `pipelineStageChanges`, and `leadScoreDistribution`

### Deal Enhancements

- **New fields:** `description`, `actualCloseDate` in the schema and API
- **Automatic recording:** `actualCloseDate` is automatically set when status changes to `WON` or `LOST`

### Business Rules

- **Customer limit (`BR-002-03`):** 100-contact limit enforced for the free plan (`422` response)
- **Password policy:** Requires at least one letter, one number, and one special character

### Notifications

- **Notification model:** In-app notification system
- **Auto-creation:** Task notifications are automatically created when `dueDate` is set
- **Endpoints:**
  - `GET /notifications` - Notification list (`unreadOnly` filter)
  - `PATCH /notifications/:id/read` - Mark as read
  - `PATCH /notifications/read-all` - Mark all as read

### Audit Logging

- **`AuditLog` model:** Comprehensive API request logging (`NFR-005`)
- **Middleware:** All API requests are logged with user, action, resource, and payload
