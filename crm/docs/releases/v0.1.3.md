# Release Notes: v0.1.3

Date: 2026-02-26

## Summary

This release implements P2 features including dashboard enhancements, notification system, and business rule enforcement.

## Highlights

### Dashboard Enhancements

- **New Metrics**: `newContactsThisWeek`, `todayRecommendationsCount` added to summary
- **GET /dashboard/activities**: Recent activities endpoint with limit parameter

### Reports

- **GET /reports/activities**: Activity statistics by type
- **Weekly/Monthly Reports**: Added `successRate`, `pipelineStageChanges`, `leadScoreDistribution`

### Deal Enhancements

- **New Fields**: `description`, `actualCloseDate` in schema and API
- **Auto-recording**: `actualCloseDate` automatically set when status changes to WON/LOST

### Business Rules

- **Customer Limit (BR-002-03)**: 100 customer limit enforced for free plan (422 response)
- **Password Policy**: Requires at least one letter, number, and special character

### Notifications

- **Notification Model**: In-app notification system
- **Auto-creation**: Task notifications automatically created when dueDate is set
- **Endpoints**:
  - GET `/notifications` - Notification list (unreadOnly filter)
  - PATCH `/notifications/:id/read` - Mark as read
  - PATCH `/notifications/read-all` - Mark all as read

### Audit Logging

- **AuditLog Model**: Comprehensive API request logging (NFR-005)
- **Middleware**: All API requests logged with user, action, resource, payload
